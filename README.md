# 断桥铝门窗用料优化驾驶舱

这是一个本地静态 Web App，用于门窗店按产品类型管理尺寸图、切割需求、余料库存和整料采购订单。

## 核心逻辑

- 材料设置里的“整料长度”是商店出售的固定长度，比如 2000cm。
- 尺寸图会生成内部切割需求，比如 150cm、180cm。
- 程序根据切割需求、切割损耗和已有余料，计算最少需要向商店购买几根整料。
- 采购订单只显示整根材料：材料、整料长度、采购根数、单价、小计。
- 具体每根料怎么切，在优化结果中查看。

## 当前功能

- 顶部驾驶舱汇总全部产品的预计采购成本、采购整料根数、可复用余料、废料和材料利用率。
- 支持四个业务页面：窗户、门、纱窗、纱门。
- 当前页面只显示本类型和“全部”通用材料。
- 支持常规模板和自定义分格。
- 支持设置每一列宽度、每一行高度，并在图纸上标注每个小格尺寸。
- 支持关闭某个小格，用于表达简单异形、空缺或不生成扇/网料的位置。
- 重复点击“生成本页采购订单”不会重复叠加。
- 可保存本页产生的可复用余料。

## 尺寸图生成规则

- 主框料：总宽 2 根，总高 2 根。
- 竖向分隔料：按每行高度分段生成。
- 横向分隔料：按每列宽度分段生成。
- 扇/网料：当前不自动生成；尺寸图默认只生成主框料和分隔/中挺料，扇料、压线、锁边等可后续按实际生产规则扩展。

实际生产中的扣尺、搭接、轨道、压线、锁边、上下滑等规则，可以继续在 `app.js` 的 `designPieces()` 中扩展。
